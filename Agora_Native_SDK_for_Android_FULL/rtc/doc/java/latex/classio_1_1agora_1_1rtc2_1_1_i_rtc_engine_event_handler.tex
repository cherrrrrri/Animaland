\doxysubsection{Detailed Description}
Callback methods.

The S\+DK uses the \mbox{\hyperlink{classio_1_1agora_1_1rtc2_1_1_i_rtc_engine_event_handler}{I\+Rtc\+Engine\+Event\+Handler}} interface class to send callback event notifications to the application, and the application inherits the methods of this interface class to retrieve these event notifications. All methods in this interface class have their (empty) default implementations, and the application can inherit only some of the required events instead of all of them. In the callback methods, the application should avoid time-\/consuming tasks or call blocking A\+P\+Is (such as Send\+Message), otherwise, the S\+DK may not work properly.

The S\+DK triggers two independent {\ttfamily on\+Audio\+Volume\+Indication} callbacks at one time, which separately report the volume information of the local user and all the remote speakers. For more information, see the detailed parameter descriptions.

\begin{DoxyItemize}
\item If the local user calls the {\ttfamily mute\+Local\+Audio\+Stream} method, the S\+DK stops triggering the local user\textquotesingle{}s callback.
\item 20 seconds after a remote speaker calls the {\ttfamily mute\+Local\+Audio\+Stream} method, the remote speakers\textquotesingle{} callback does not include information of this remote user; 20 seconds after all remote users call the {\ttfamily mute\+Local\+Audio\+Stream} method, the S\+DK stops triggering the remote speakers\textquotesingle{} callback.
\end{DoxyItemize}
