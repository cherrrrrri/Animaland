\doxysubsection{Detailed Description}
Interfaces for Extension Provider User can implement these interfaces for providing their own media node implementations to S\+DK. Please refer to \mbox{\hyperlink{classagora_1_1_ref_counted_object}{agora\+::\+Ref\+Counted\+Object}} to wrap your implementation so that it can be held by \mbox{\hyperlink{classagora_1_1agora__refptr}{agora\+::agora\+\_\+refptr}}. For example\+: class Your\+Extension\+Provide\+: public \mbox{\hyperlink{classagora_1_1rtc_1_1_i_extension_provider}{I\+Extension\+Provider}} \{ // Your Concrete implementaion \};

// Use agroa\+::\+Ref\+Counted\+Object to provide \mbox{\hyperlink{classagora_1_1_ref_count_interface}{Ref\+Count\+Interface}} implementation for you implementation, // intantiate and wrap it with \mbox{\hyperlink{classagora_1_1agora__refptr}{agora\+\_\+refptr}}.

agora\+\_\+refptr$<$\+I\+Extension\+Provider$>$ provider = new Ref\+Counted\+Object$<$\+Your\+Extension\+Provide$>$(\+Arg1, Arg2, ...);

You can instantiate your Audio\+Filter/\+Video\+Filter/\+Video\+Sink in the same way.
